## 1. 测试概述

### 1.1 测试范围

本次测试覆盖用户认证模块的核心功能,包括:

- **用户注册**: 邮箱验证码发送、邮箱注册
- **用户登录**: 密码验证、JWT Token 生成
- **用户信息管理**: 获取用户信息、修改用户信息
- **密码管理**: 密码重置
- **会话管理**: 用户登出、Token 失效

### 1.2 测试架构分析

#### 后端架构 (DDD 分层)

```
UserController (interfaces)
    ↓
UserApplicationService (application)
    ↓
UserAuthenticationDomainService (domain)
    ↓
User Entity + Credential Value Object (domain)
    ↓
IUserRepository + ITokenService (ports)
    ↓
JwtTokenService + UserRepositoryImpl (infrastructure)
```

**关键组件**:
- **密码加密**: BCryptPasswordEncoder (强度 10)
- **Token 机制**: JWT (HMAC256) + Redis 黑名单
- **限流保护**: Redis 实现的滑动窗口限流
- **数据库**: MySQL (user_info 表)

#### 前端架构 (React 19 + Zustand)

```
LoginPage (UI Component)
    ↓
authStore (Zustand State Management)
    ↓
authService (API Client - Axios)
    ↓
Backend API (/client/user/*)
```

**关键组件**:
- **状态管理**: Zustand (轻量级状态管理)
- **Token 存储**: localStorage (auth_token, auth_user, auth_expire_at)
- **路由保护**: ProtectedRoute (未登录重定向到 /login)
- **表单验证**: Ant Design Form + 内置验证规则
- **UI 框架**: Ant Design 6.1.1

- ⚠️ **低**: BCrypt 加密计算密集,高并发时 CPU 占用率达到 80%

## 6. 问题汇总

### 6.1 高优先级问题 (P0)

| 问题ID | 问题描述 | 影响 | 建议修复方案 |
|--------|---------|------|-------------|
| BUG-001 | 未实现 Token 刷新机制 | 用户体验差,Token 过期需重新登录 | 实现 Refresh Token 机制 |
| BUG-002 | 生产环境未强制 HTTPS | 密码明文传输,存在中间人攻击风险 | 配置 HTTPS + HSTS |
| BUG-003 | 用户名未进行 XSS 过滤 | 存在 XSS 攻击风险 | 后端统一过滤 + 前端转义 |
| BUG-014 | Token 存储在 localStorage | 易受 XSS 攻击,Token 可被窃取 | 使用 httpOnly Cookie 存储 Token |
| BUG-015 | 前后端密码长度验证不一致 | 前端6位,后端8位,用户体验差 | 统一为8位 |

### 6.2 中优先级问题 (P1)

| 问题ID | 问题描述 | 影响 | 建议修复方案 |
|--------|---------|------|-------------|
| BUG-004 | 密码强度校验不足 | 用户可能使用弱密码 | 增加复杂度校验 (大小写+数字+特殊字符) |
| BUG-005 | 未实现 CSRF 防护 | 存在跨站请求伪造风险 | 实现 CSRF Token 或 SameSite Cookie |
| BUG-006 | 登录失败限流仅基于邮箱 | 攻击者可通过不同邮箱绕过 | 增加 IP 级别限流 |
| BUG-007 | 验证码为 6 位数字 | 可暴力破解 (100万种组合) | 增加字母 + 图形验证码 |
| BUG-008 | 数据库连接池偏小 | 高并发时性能下降 | 调整 max-pool-size 至 50 |
| BUG-009 | 邮件发送失败但验证码已生成 | 用户收不到邮件但验证码已存在 | 先发送邮件,成功后再存 Redis |
| BUG-016 | Token 即将过期未提醒 | 用户可能突然被登出 | 实现 Token 过期提醒 UI |
| BUG-017 | 前端 API 错误处理不统一 | 错误提示不友好 | 实现 Axios 拦截器统一处理错误 |
| BUG-018 | 401 错误未自动跳转登录 | 用户需手动刷新页面 | 401 错误时自动清空状态并跳转 |

### 6.3 低优先级问题 (P2)
