## 一、认证模块 (Auth)

### TC-AUTH-01: 邮箱注册
- 前置：未注册的邮箱
- 步骤：
  1. 访问 /register
  2. 输入邮箱、发送验证码
  3. 输入验证码、密码、确认密码
  4. 点击注册
- 预期：注册成功，自动登录跳转到 /dashboard

### TC-AUTH-02: 邮箱登录
- 步骤：
  1. 访问 /login
  2. 输入邮箱和密码
  3. 点击登录
- 预期：登录成功，跳转 /dashboard，localStorage 存有 accessToken

### TC-AUTH-03: 登录失败 - 错误密码
- 步骤：输入正确邮箱 + 错误密码，点击登录
- 预期：提示"邮箱或密码错误"，不跳转

### TC-AUTH-04: 登录失败 - 空字段
- 步骤：不填任何字段，点击登录
- 预期：表单校验提示必填

### TC-AUTH-05: 忘记密码
- 步骤：
  1. 访问 /forgot-password
  2. 输入已注册邮箱，发送验证码
  3. 输入验证码和新密码
  4. 提交
- 预期：密码重置成功，可用新密码登录

### TC-AUTH-06: Token 刷新
- 步骤：accessToken 过期后发起请求
- 预期：自动刷新 token，请求不中断

### TC-AUTH-07: 登出
- 步骤：点击登出按钮
- 预期：清除 token，跳转 /login

### TC-AUTH-08: 未登录访问受保护页面
- 步骤：未登录状态直接访问 /dashboard
- 预期：重定向到 /login

## 四、工作流编辑器模块 (Workflow)

### TC-WF-01: 加载工作流
- 前置：已有 Agent
- 步骤：访问 /agents/:agentId/workflow
- 预期：正确加载工作流画布，显示已有节点和连线

### TC-WF-02: 添加节点
- 步骤：从节点模板面板拖拽/点击添加节点到画布
- 预期：节点出现在画布上，可选中编辑

### TC-WF-03: 连接节点
- 步骤：从一个节点的输出 handle 拖拽到另一个节点的输入 handle
- 预期：生成连线，连线可见

### TC-WF-04: 删除节点
- 步骤：选中节点，按 Delete 或点击删除按钮
- 预期：节点及其关联连线被删除

### TC-WF-05: 删除连线
- 步骤：选中连线，按 Delete
- 预期：连线被删除

### TC-WF-06: 编辑节点配置
- 步骤：双击/点击节点，在配置面板修改参数
- 预期：配置保存成功，节点显示更新

### TC-WF-07: 保存工作流
- 步骤：修改工作流后点击保存
- 预期：保存成功提示，刷新页面后修改仍在

### TC-WF-08: 发布工作流
- 步骤：点击发布按钮
- 预期：发布成功，Agent 状态变为已发布

### TC-WF-09: 工作流校验 - 缺少必要节点
- 步骤：删除 START 或 END 节点后尝试保存/发布
- 预期：校验失败，提示缺少必要节点

### TC-WF-10: 工作流校验 - 未连接节点
- 步骤：添加孤立节点后尝试发布
- 预期：校验警告，提示有未连接的节点

### TC-WF-11: 条件分支节点
- 步骤：添加条件节点，配置多个分支条件
- 预期：每个分支有独立的输出 handle，可分别连接不同节点

### TC-WF-12: 画布缩放和拖拽
- 步骤：鼠标滚轮缩放、拖拽画布
- 预期：画布平滑缩放和移动

## 五、对话模块 (Chat)

### TC-CHAT-01: 创建对话

- 步骤：在对话框输入消息，点击发送

### TC-CHAT-05: 多轮对话
- 步骤：连续发送多条消息
- 预期：每轮对话都能正确响应，上下文连贯

### TC-CHAT-06: 人工审核暂停
- 步骤：工作流中包含人工审核节点，触发执行
- 预期：执行到审核节点时暂停，提示等待审核

### TC-CHAT-07: 对话列表

## 十一、跨模块集成场景

### TC-INT-01: 完整工作流：创建 Agent → 编辑工作流 → 发布 → 对话
- 步骤：
  1. 创建新 Agent
  2. 进入工作流编辑器，搭建 START → LLM → END 流程
  3. 配置 LLM 节点（选择模型）
  4. 保存并发布
  5. 进入 Chat 页面，选择该 Agent 对话
  6. 发送消息，验证回复
- 预期：全流程通畅，对话正常响应

### TC-INT-02: 知识库关联 Agent
- 步骤：
  1. 创建知识库并上传文档
  2. 在 Agent 工作流中添加知识检索节点
  3. 关联知识库
  4. 发布并对话，提问文档相关内容
- 预期：Agent 能基于知识库内容回答

### TC-INT-03: 人工审核完整流程
- 步骤：
  1. 在工作流中添加人工审核节点
  2. 发布 Agent
  3. 发起对话触发审核
  4. 在 /reviews 页面通过审核
  5. 回到对话页面查看后续输出
- 预期：审核通过后工作流继续执行，对话收到完整回复

### TC-INT-04: Swarm 多 Agent 协作
- 步骤：
  1. 创建工作空间
  2. 创建多个不同角色的 Agent
  3. 发送任务消息
  4. 观察 Agent 间的协作和消息传递
- 预期：多 Agent 按角色分工协作

## 十二、异常与边界场景

### TC-ERR-01: 后端服务不可用
- 步骤：停止后端服务，前端发起请求
- 预期：显示友好的错误提示，不白屏

### TC-ERR-02: 网络断开恢复
- 步骤：对话过程中断网，恢复后重试
- 预期：能重新连接，不丢失已有数据

### TC-ERR-03: 并发操作
- 步骤：快速连续点击创建按钮
- 预期：不产生重复数据，按钮有防重复点击

### TC-ERR-04: 大文件上传
- 步骤：上传超大文件到知识库
- 预期：有文件大小限制提示或正确处理

### TC-ERR-05: 长文本输入
- 步骤：在对话框输入超长文本
- 预期：正常发送或有长度限制提示

### TC-ERR-06: SSE 连接断开重连
- 步骤：对话流式输出过程中网络波动
- 预期：自动重连或提示用户刷新
