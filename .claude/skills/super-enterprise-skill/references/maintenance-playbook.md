# 业务分层 Skill 维护手册

## 第一规则

当目标工作区 skill 存在 `logs/YYYY-MM.md` 时，修改规则或 SOP 前先读取最近日志。
日志可能包含用户纠正和近期决策，它们会覆盖更旧的 SOP 文本。

## 判断一次变更的归属

使用这张归属表：

| Change | Target |
|---|---|
| 新的带日期事实、一次性工作、近期决策 | `logs/YYYY-MM.md` |
| 可复用业务流程 | `references/<business-domain>/<workflow>.md` |
| 业务域路由变化 | `references/<business-domain>/index.md` |
| 跨域路由或边界变化 | 根 `SKILL.md` 路由表 |
| 跨工作区安全规则 | 根 `SKILL.md` |
| 事故签名或重复失败 | `references/incidents/...` |
| 脚本输入/输出/env/副作用变化 | `scripts/<family>/README.md` |
| 专用 agent prompt/config 变化 | `agents/<agent-name>/...` |
| 触发质量问题 | 根 frontmatter `description` |
| 支撑仓库/服务/域名/数据库事实 | 相关业务域上下文文件 |

优先选择最窄且持久的归属。不要为了局部事实更新根文件。

## 泛化规则

当任务是为另一家公司或项目集合初始化、维护工作区 skill 时，不要把源工作区事实复制到目标。

写入任何具体仓库/服务/域名/数据库/业务域标签前，先回答这个问题：

`这是在目标工作区发现的，还是只来自源示例？`

如果它只来自源示例，保留占位符或泛化标签，不要复制它。

## 新增业务域

1. 识别用户措辞、业务动作、仓库、服务、域名、数据库、脚本和安全等级。
2. 创建 `references/<business-domain>/index.md`。
3. 只有在已知真实工作流细节时，才添加具体 SOP 文件。
4. 添加业务域地图行和 `Meet -> See` 路由行。
5. 验证所有被引用路径。
7. 验证业务域标签来自目标工作区，而不是源示例。

## 新增业务工作流 SOP

1. 使用 `sop-taxonomy.md` 分类 SOP。
2. 判断哪个业务域拥有该动作。
3. 创建带明确必要输入、停止条件、安全和验证的 SOP 骨架。
4. 从业务域 `index.md` 链接它。
5. 只有当用户很可能直接询问该工作流时，才从根路由表链接它。
6. 将支撑仓库/服务/数据库记录为事实，而不是单独的一等顶层项目路由。
7. 标记缺失事实，不要编造生产细节。

## 去源化草稿

如果生成草稿意外继承了源工作区标签：

1. 移除源专属工作区名称。
2. 用通用占位符或目标已确认名称替换源业务域名称。
3. 除非目标已确认，否则移除源仓库/服务/域名/数据库。
4. 重新执行路由完整性和可移植性验证。

## 更新支撑服务上下文

当业务 SOP 需要仓库/服务事实，但这些事实本身不定义业务工作流时，使用支撑上下文文件。

示例：

- 账单工作流需要多个服务的表名。
- 账号池工作流需要导入脚本位置。
- 基础设施工作流需要 compose/Caddy/服务注册事实。

除非用户明确想要项目导向路由，否则不要为这些事实创建一等项目路由。

## 更新安全规则

只有当安全变化跨业务域适用时，才放在根级。同时在日志中添加带日期说明，解释规则为什么变化。

示例：

- 数据库写入确认规则跨工作区适用 -> 根。
- 某个账号导入 dry-run 规则只适用于账号 -> accounts SOP 或脚本 README。
- 前端浏览器验证要求适用于许多 web 工作流 -> 根。

## 重命名或移动业务 SOP

1. 更新根路由表、域索引和交叉链接。
2. 完成前搜索旧路径。
3. 如果重命名影响触发措辞或路由，添加简短说明。

## 废弃业务域或 SOP

默认不要静默删除路由。

1. 将条目标记为 deprecated。
2. 指向替代路由。
3. 移除或收窄执行能力。
4. 保留足够历史，使 agent 能理解旧用户措辞。

## 冲突处理

出现路由冲突时：

- 优先用户点名的 SOP。
- 优先更具体的业务动作。
- 高风险执行优先更严格安全等级。
- 如果仓库/服务信号指向一个方向、业务动作指向另一个方向，先按业务动作路由，再加载服务上下文。
- 如果两个 SOP 都声称拥有同一流程，记录冲突并添加路由规则。

不要静默合并两个安全假设不同的 SOP。

## 维护输出

维护工作结束时输出：

- 变更文件；
- 变更的业务路由条目；
- 安全边界变化；
- 已执行验证；
- 发现的过期路径或冲突；
- 未解决问题。
