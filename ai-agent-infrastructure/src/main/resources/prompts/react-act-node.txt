## 任务要求

请使用指定的工具和参数执行当前步骤,并如实报告执行结果。

### 执行原则
1. **严格执行**: 必须使用指定的工具和参数,不要擅自更改
2. **真实反馈**: 如实报告工具的实际输出,不要编造或美化
3. **错误处理**: 如果工具调用失败,详细说明失败原因
4. **完整信息**: 确保提取工具返回的所有关键信息

### 状态定义
- **SUCCESS**: 工具调用成功,获得了预期的信息
- **FAILED**: 工具调用失败,或返回结果异常

---

## 输出格式要求

**严格按照以下 JSON 格式输出,不要添加任何解释或其他内容:**

```json
{
  "status": "SUCCESS 或 FAILED",
  "toolResult": "工具返回的实际结果(如果成功)",
  "errorMessage": "错误信息(如果失败)"
}
```

---

## 输出示例

### 成功示例
```json
{
  "status": "SUCCESS",
  "toolResult": "北京今日天气:晴,温度 15-25℃,空气质量:良(AQI 85),东南风 3-4 级,湿度 45%",
  "errorMessage": null
}
```

### 失败示例
```json
{
  "status": "FAILED",
  "toolResult": null,
  "errorMessage": "网络请求超时,无法获取天气数据"
}
```

---

**重要提醒:**
- 只输出 JSON 对象,不要有任何前缀或后缀
- 确保 JSON 格式正确,可被解析
- status 字段只能是 "SUCCESS" 或 "FAILED"
- 如果成功,toolResult 不能为空
- 如果失败,errorMessage 必须说明原因
