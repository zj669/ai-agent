## 1. 用户模块 (User Module)

### TC-USER-001: 用户注册 - 正常流程
- **前置条件**: 用户未登录，使用未注册的邮箱
- **测试步骤**:
  1. 访问 http://localhost:5173/login
  2. 点击"注册"标签页
  3. 输入有效邮箱地址（如 test@example.com）
  4. 点击"发送验证码"按钮
  5. 输入收到的验证码
  6. 输入密码（至少8位）
  7. 输入用户名（可选）
  8. 点击"注册"按钮
- **预期结果**:
  - 验证码发送成功，提示"验证码已发送,5分钟内有效,请查收邮件"
  - 注册成功后自动登录，跳转到 Dashboard 页面
- **涉及API**:
  - POST `/client/user/email/sendCode`
  - POST `/client/user/email/register`

### TC-USER-002: 用户注册 - 邮箱已被注册
- **前置条件**: 使用已注册的邮箱
- **测试步骤**:
  1. 访问 http://localhost:5173/login
  2. 点击"注册"标签页
  3. 输入已注册的邮箱地址
  4. 发送验证码并输入
  5. 输入密码和用户名
  6. 点击"注册"按钮
- **预期结果**: 显示错误提示"该邮箱已被注册"
- **涉及API**: POST `/client/user/email/register`

### TC-USER-003: 用户登录 - 正常流程
- **前置条件**: 用户已注册
- **测试步骤**:
  1. 访问 http://localhost:5173/login
  2. 在"登录"标签页输入邮箱
  3. 输入正确密码
  4. 点击"登录"按钮
- **预期结果**:
  - 登录成功，提示"登录成功"
  - 跳转到 Dashboard 页面
  - 页面右上角显示用户信息
- **涉及API**: POST `/client/user/login`

### TC-USER-004: 用户登录 - 密码错误
- **前置条件**: 用户已注册
- **测试步骤**:
  1. 访问 http://localhost:5173/login
  2. 输入正确邮箱
  3. 输入错误密码
  4. 点击"登录"按钮
- **预期结果**: 显示错误提示"邮箱或密码错误"
- **涉及API**: POST `/client/user/login`

### TC-USER-005: 重置密码 - 正常流程
- **前置条件**: 用户已注册
- **测试步骤**:
  1. 访问 http://localhost:5173/login
  2. 点击"忘记密码？"链接
  3. 输入注册邮箱
  4. 点击"发送验证码"
  5. 输入验证码
  6. 输入新密码（至少6位）
  7. 确认新密码
  8. 点击"重置密码"按钮
- **预期结果**:
  - 密码重置成功，提示"密码重置成功，请使用新密码登录"
  - 跳转到登录页面
- **涉及API**:
  - POST `/client/user/email/sendCode`
  - POST `/client/user/resetPassword`

## 2. Agent 模块 (Agent Module)

### TC-AGENT-001: 创建 Agent - 正常流程
- **前置条件**: 用户已登录
- **测试步骤**:
  1. 访问 http://localhost:5173/agents
  2. 点击"创建 Agent"按钮
  3. 输入 Agent 名称（如"测试助手"）
  4. 输入描述（如"这是一个测试用的 AI 助手"）
  5. 输入图标（如"🤖"）
  6. 点击"保存"按钮
- **预期结果**:
  - Agent 创建成功
  - 跳转到 Agent 列表页面
  - 新创建的 Agent 显示在列表中，状态为"草稿"
- **涉及API**: POST `/api/agent/create`

### TC-AGENT-002: 编辑 Agent - 修改基本信息
- **前置条件**: 用户已登录，存在至少一个 Agent
- **测试步骤**:
  1. 访问 http://localhost:5173/agents
  2. 点击某个 Agent 的编辑按钮（或点击名称）
  3. 修改 Agent 名称
  4. 修改描述
  5. 点击"保存"按钮
- **预期结果**:
  - 保存成功
  - 返回列表页面后显示更新后的信息
- **涉及API**:
  - GET `/api/agent/{id}`
  - POST `/api/agent/update`

### TC-AGENT-003: 发布 Agent
- **前置条件**: 用户已登录，存在状态为"草稿"的 Agent
- **测试步骤**:
  1. 访问 http://localhost:5173/agents
  2. 找到状态为"草稿"的 Agent
  3. 点击"发布"按钮（火箭图标）
  4. 确认发布操作
- **预期结果**:
  - Agent 状态变为"已发布"
  - 发布按钮消失
- **涉及API**: POST `/api/agent/publish`

### TC-AGENT-004: 删除 Agent
- **前置条件**: 用户已登录，存在至少一个 Agent
- **测试步骤**:
  1. 访问 http://localhost:5173/agents
  2. 点击某个 Agent 的删除按钮
  3. 确认删除操作
- **预期结果**:
  - Agent 从列表中移除
  - 显示删除成功提示
- **涉及API**: DELETE `/api/agent/{id}/force`

### TC-AGENT-005: 搜索 Agent
- **前置条件**: 用户已登录，存在多个 Agent
- **测试步骤**:
  1. 访问 http://localhost:5173/agents
  2. 在搜索框中输入关键词
  3. 观察列表变化
- **预期结果**: 列表只显示名称或描述包含关键词的 Agent
- **涉及API**: GET `/api/agent/list`（前端过滤）

## 3. 工作流模块 (Workflow Module)

### TC-WF-001: 进入工作流编辑器
- **前置条件**: 用户已登录，存在至少一个 Agent
- **测试步骤**:
  1. 访问 http://localhost:5173/agents
  2. 点击某个 Agent 的编辑按钮
  3. 进入工作流编辑器页面
- **预期结果**:
  - 显示工作流编辑器界面
  - 左侧显示节点面板
  - 中间显示画布区域
  - 如果 Agent 已有工作流配置，则加载显示
- **涉及API**: GET `/api/agent/{id}`

### TC-WF-002: 添加节点到画布
- **前置条件**: 用户已进入工作流编辑器
- **测试步骤**:
  1. 从左侧节点面板拖拽"开始节点"到画布
  2. 拖拽"LLM 节点"到画布
  3. 拖拽"结束节点"到画布
- **预期结果**:
  - 节点成功添加到画布
  - 底部状态栏显示节点数量更新
  - 保存状态变为"未保存"
- **涉及API**: 无（前端操作）

### TC-WF-003: 连接节点
- **前置条件**: 画布上已有多个节点
- **测试步骤**:
  1. 从开始节点的输出端口拖拽连线到 LLM 节点的输入端口
  2. 从 LLM 节点的输出端口拖拽连线到结束节点的输入端口
- **预期结果**:
  - 节点之间成功建立连接
  - 连接线显示为平滑曲线
  - 底部状态栏显示连接数量更新
- **涉及API**: 无（前端操作）

### TC-WF-004: 配置 LLM 节点属性
- **前置条件**: 画布上已有 LLM 节点
- **测试步骤**:
  1. 点击 LLM 节点
  2. 在右侧属性面板中配置：
     - 节点名称
     - 模型选择
     - 系统提示词
     - 温度参数
  3. 关闭属性面板
- **预期结果**:
  - 属性面板正确显示节点配置
  - 修改后节点显示更新的名称
- **涉及API**: 无（前端操作）

### TC-WF-005: 保存工作流
- **前置条件**: 画布上已有完整的工作流（开始→LLM→结束）
- **测试步骤**:
  1. 点击顶部工具栏的"保存"按钮
  2. 或使用快捷键 Ctrl+S
- **预期结果**:
  - 显示"保存中..."状态
  - 保存成功后显示"已保存"
  - 提示"工作流已保存"
- **涉及API**: POST `/api/agent/update`

### TC-WF-006: 执行工作流 - 标准模式
- **前置条件**: 工作流已保存，包含完整的节点连接
- **测试步骤**:
  1. 点击顶部工具栏的"运行"按钮
  2. 在弹出的对话框中：
     - 输入会话 ID
     - 输入输入参数（JSON 格式，如 `{"query": "你好"}`）
     - 选择"标准模式"
  3. 点击"开始执行"
- **预期结果**:
  - 工作流开始执行
  - 右下角显示执行日志面板
  - 日志实时更新节点执行状态
  - 执行完成后显示结果
- **涉及API**: POST `/api/workflow/execution/start`（SSE）

### TC-WF-007: 停止工作流执行
- **前置条件**: 工作流正在执行中
- **测试步骤**:
  1. 点击顶部工具栏的"停止"按钮
- **预期结果**:
  - 工作流执行停止
  - 显示"工作流已停止"提示
  - 执行日志显示停止状态
- **涉及API**: POST `/api/workflow/execution/stop`

### TC-WF-008: 导出工作流
- **前置条件**: 画布上已有工作流配置
- **测试步骤**:
  1. 点击顶部工具栏的更多按钮（三个点）
  2. 选择"导出工作流"
- **预期结果**:
  - 浏览器下载 JSON 文件
  - 文件名格式为 `{工作流名称}_{时间戳}.json`
- **涉及API**: 无（前端操作）

### TC-WF-009: 导入工作流
- **前置条件**: 有之前导出的工作流 JSON 文件
- **测试步骤**:
  1. 点击顶部工具栏的更多按钮
  2. 选择"导入工作流"
  3. 选择 JSON 文件
- **预期结果**:
  - 工作流配置加载到画布
  - 显示"工作流已导入"提示
- **涉及API**: 无（前端操作）

### TC-WF-010: 清空画布
- **前置条件**: 画布上已有节点
- **测试步骤**:
  1. 点击顶部工具栏的更多按钮
  2. 选择"清空画布"
  3. 确认清空操作
- **预期结果**:
  - 画布上所有节点和连接被清除
  - 显示空状态提示
- **涉及API**: 无（前端操作）

## 4. 对话模块 (Chat Module)

### TC-CHAT-001: 创建新会话
- **前置条件**: 用户已登录，存在已发布的 Agent
- **测试步骤**:
  1. 访问 http://localhost:5173/chat
  2. 从下拉框选择一个 Agent
  3. 点击"新建"按钮
- **预期结果**:
  - 创建新会话成功
  - 会话列表中显示新会话
  - 右侧对话区域显示空状态
- **涉及API**: POST `/api/chat/conversations`

### TC-CHAT-002: 发送消息
- **前置条件**: 已创建会话
- **测试步骤**:
  1. 在输入框中输入消息（如"你好"）
  2. 点击"发送"按钮或按 Enter 键
- **预期结果**:
  - 用户消息显示在对话区域
  - AI 开始生成回复
  - AI 回复显示在对话区域
- **涉及API**:
  - POST `/api/workflow/execution/start`（触发工作流）
  - 或其他消息发送 API

### TC-CHAT-003: 切换会话
- **前置条件**: 存在多个会话
- **测试步骤**:
  1. 在会话列表中点击另一个会话
- **预期结果**:
  - 右侧对话区域显示该会话的历史消息
  - 当前选中的会话高亮显示
- **涉及API**: GET `/api/chat/conversations/{conversationId}/messages`

### TC-CHAT-004: 删除会话
- **前置条件**: 存在至少一个会话
- **测试步骤**:
  1. 点击会话右侧的删除按钮
  2. 确认删除操作
- **预期结果**:
  - 会话从列表中移除
  - 如果删除的是当前会话，对话区域显示空状态
- **涉及API**: DELETE `/api/chat/conversations/{conversationId}`

## 7. 边界情况和异常测试

### TC-EDGE-001: 登录失败次数过多
- **前置条件**: 无
- **测试步骤**:
  1. 连续多次使用错误密码登录同一账号
- **预期结果**: 显示"登录失败次数过多,账号已被锁定15分钟,请稍后再试"
- **涉及API**: POST `/client/user/login`

### TC-EDGE-002: Token 过期自动刷新
- **前置条件**: 用户已登录，Token 即将过期
- **测试步骤**:
  1. 等待 Token 过期
  2. 执行任意需要认证的操作
- **预期结果**:
  - 系统自动刷新 Token
  - 操作正常完成
  - 用户无感知
- **涉及API**: POST `/client/user/refresh`

### TC-EDGE-003: 验证码发送频率限制
- **前置条件**: 无
- **测试步骤**:
  1. 在注册或重置密码页面
  2. 连续快速点击"发送验证码"按钮
- **预期结果**:
  - 第一次发送成功
  - 后续点击显示倒计时
  - 倒计时结束前无法再次发送
- **涉及API**: POST `/client/user/email/sendCode`

### TC-EDGE-004: 工作流执行超时
- **前置条件**: 工作流包含耗时较长的节点
- **测试步骤**:
  1. 执行一个包含复杂 LLM 调用的工作流
  2. 等待执行
- **预期结果**:
  - 如果超时，显示超时错误
  - 执行状态变为失败
- **涉及API**: POST `/api/workflow/execution/start`

### TC-EDGE-005: 大文件上传
- **前置条件**: 已创建知识库
- **测试步骤**:
  1. 尝试上传一个较大的文档（如 10MB+）
- **预期结果**:
  - 显示上传进度
  - 如果超过限制，显示文件过大错误
  - 如果在限制内，正常上传和处理
- **涉及API**: POST `/api/knowledge/document/upload`

## 测试环境要求

### 后端服务
- Spring Boot 应用运行在 8080 端口
- MySQL 数据库运行在 13306 端口
- Redis 运行在 6379 端口
- Milvus 运行在 19530 端口
