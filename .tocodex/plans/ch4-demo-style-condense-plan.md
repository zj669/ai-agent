# 第四章参照 demo 精简改写计划

## 目标文件
- `论文/chapter/ch4_implementation.tex`

## 参考 demo 写法结论
1. 每个功能模块采用“实现过程 → 关键代码 → 界面展示”的三段式。
2. 实现过程多为短自然段，不展开过细的“第一步、第二步”列表。
3. 关键代码只展示核心片段，通常控制在 10~20 行左右。
4. 代码用于证明实现思路，不应把完整业务方法全部搬进论文。

## 本次修改原则
1. 保持当前五模块顺序不变：用户认证、Agent 管理、工作流调度、人工审核、知识库。
2. 保留 SSE 作为实时交互支撑模块。
3. 不新增大段内容，只对第四章进行“瘦身”和“论文风格化”。
4. 所有关键代码编号保持从 `代码4-1` 到 `代码4-15` 连续。
5. 保留必要核心代码，不展示完整异常处理、对象构造、批量循环和重复校验细节。
