\chapter{结论与展望}

\section{研究结论}

本文设计并实现了一套\textbf{基于 Spring Boot 的 AI Agent 智能体编排系统}，
面向 Java 生态中智能体应用开发缺乏原生编排平台的问题，围绕 Agent 配置、流程执行、
人工审核、知识增强与访问安全五个方面完成了系统设计与实现，主要结论如下：

\textbf{（1）实现了 Agent 管理与可视化编排能力。}
系统支持通过可视化画布完成节点拖拽、参数配置、边连接与流程保存，
并以 \texttt{graphJson} 形式持久化编排结果。围绕 Agent 生命周期，系统进一步实现了
创建、更新、发布、版本查询与回滚机制，使工作流定义具备可配置、可追溯、可复用的工程化特征。

\textbf{（2）构建了事件驱动的 DAG 工作流调度执行机制。}
系统借鉴图结构调度思想，基于拓扑排序实现了"就绪节点驱动"的异步调度策略，
支持无依赖节点的自动并行执行、条件分支与路径剪枝。通过 \texttt{Execution} 聚合根
统一维护节点状态、上下文数据和执行推进逻辑，保证了工作流执行过程的可控性与可维护性。

\textbf{（3）实现了人工检查点与工作流恢复机制。}
系统支持 BEFORE\_EXECUTION 和 AFTER\_EXECUTION 两种审核触发阶段，
审核人可查看执行上下文并对中间结果进行修改后放行，也可拒绝并终止流程。
同时，系统通过 \texttt{reviewedNodes} 防止重复暂停，结合 Redis 分布式锁与乐观锁版本控制，
保障多人并发审批场景下的状态一致性与审计可追踪性。

\textbf{（4）实现了知识库与检索增强链路。}
系统支持知识数据集管理、文档上传、异步解析、文本分块与向量化入库，
并借助 Milvus 向量数据库提供语义检索能力。在工作流运行过程中，
检索结果可作为上下文注入 LLM 节点，从而提升模型回答与业务知识之间的相关性。

\textbf{（5）实现了用户认证与系统安全支撑能力。}
系统完成了邮箱注册、登录、验证码校验、密码找回等基础身份能力，
并通过 BCrypt 密码加密、验证码发送频率限制、登录失败限流与 JWT 令牌认证等机制，
为 Agent 管理、知识库管理和工作流执行等功能提供统一的访问控制基础。

综合来看，本文实现的系统已经打通了“可视化编排 → 工作流执行 → 人工审核 → 知识增强 → 安全访问”的核心闭环，
具备面向 MVP 演示与业务验证的工程化基础，也为 Java 生态下 AI Agent 系统的落地提供了可复用的实现思路。

\section{研究不足}

受时间和精力限制，本文工作仍存在以下不足：

\textbf{（1）Agent 编排能力仍以单工作流为主。}
当前系统已支持可视化编排与版本管理，但在复杂子流程复用、组合模板沉淀、跨 Agent 协同编排等方面仍有提升空间。

\textbf{（2）知识增强链路仍可进一步优化。}
虽然系统已经实现文档分块、向量化与语义检索，但在分块策略自适应、检索重排、知识命中解释性等方面仍较为基础。

\textbf{（3）人工审核交互体验有待提升。}
当前审批页面主要以结构化文本和 JSON 内容展示节点输入输出，对非技术用户而言仍不够友好，
缺乏富文本、表格、图片等多类型内容的差异化展示能力。

\textbf{（4）性能测试与高可用验证范围有限。}
当前验证重点放在核心链路可用性，尚未在更高并发规模、服务重启恢复、分布式部署等场景下进行充分测试。

\section{未来工作展望}

基于上述不足，后续研究和工程优化方向包括：

\textbf{（1）增强 Agent 编排的复用与组合能力。}
可进一步引入子工作流、流程模板、并行聚合节点等能力，提升复杂业务场景下的编排表达力。

\textbf{（2）完善知识检索增强效果。}
可探索更细粒度的分块策略、召回重排模型、知识来源可解释展示等方法，
提升知识命中质量与生成结果稳定性。

\textbf{（3）优化人工审核交互与通知机制。}
未来可结合字段级渲染、差异对比视图、邮件或即时消息通知机制，
提升人工审核节点在企业协作场景中的可用性与响应效率。

\textbf{（4）加强系统性能与高可用能力。}
后续可围绕更大规模并发调度、SSE 长连接稳定性、执行断点恢复与分布式部署进行优化，
进一步提升系统在生产环境中的稳定性。

\textbf{（5）拓展多智能体协作与工具生态。}
在现有单 Agent 编排能力基础上，可进一步探索多智能体协同机制、更加丰富的工具接入方式，
以及与 Spring AI 新特性的更深度整合，从而扩展系统的适用范围与智能体协作能力。
