\chapter{绪论}

\section{选题背景及意义}

随着大语言模型（Large Language Model，LLM）推理能力、工具调用能力和上下文理解能力的快速发展，人工智能应用正从传统的问答式交互逐步扩展到能够理解任务目标、规划执行步骤并调用外部工具完成复杂任务的智能体（AI Agent）形态。与单轮问答系统相比，AI Agent 更强调任务拆解、流程控制、状态记忆和外部环境交互，能够在一定程度上把大模型能力嵌入到真实业务流程之中。围绕 AI Agent 的开发、编排和部署，业界已经出现了 Dify、Coze 等拖拉拽式低代码平台，也出现了 LangGraph、AutoGen\upcite{wu2023autogen} 等面向开发者的编排框架，这些工具降低了智能体应用从原型验证走向工程实践的门槛。

然而，从企业级后端系统建设的角度看，现有 Agent 编排能力仍存在两方面不足。首先，较成熟的 Agent 编排工具和开发生态更多集中在 Python、JavaScript 或独立低代码平台中，而许多企业核心业务系统仍长期采用 Java 和 Spring Boot 技术体系构建。Java 生态并非缺少大模型接入能力，相关框架已经提供了模型调用、提示词模板、结构化输出和函数调用等基础能力，但面向业务流程的可视化 Agent 编排、图结构运行和状态恢复实践相对较少，开发者在将 Agent 工作流嵌入既有 Java 后端系统时仍需要进行较多工程化设计。

其次，现有拖拉拽式低代码平台通常以“配置完成后自动执行”为主要范式，工作流一旦启动便从起点连续运行到终点，更适合自动化程度较高、风险较低的任务场景。在涉及内容审核、业务审批、执行结果确认或关键决策的场景中，系统需要在特定节点暂停执行，允许人工查看上下文、校验中间结果、进行必要修改，并在审批通过后恢复流程或在审批拒绝后终止流程。若缺少这种人工审核与流程恢复机制，智能体虽然能够自动执行任务，却难以满足企业场景对过程可控、结果可校验和操作可追溯的要求。

针对上述问题，本课题设计并实现一套基于 Spring Boot 的 AI Agent 智能体编排系统。系统以 Java 后端工程实践为基础，将可视化工作流编排、DAG 调度执行、人工审核检查点、知识库检索增强和用户认证安全能力结合起来，使应用构建者能够通过前端画布配置 Agent 工作流，使业务使用者能够触发已发布流程，并在关键节点通过人工审核参与执行过程。该课题的研究意义在于：一方面探索 Java 技术栈下智能体可视化编排系统的实现路径，另一方面将人工审核机制嵌入 Agent 执行链路，为需要“自动执行 + 人工把关”的业务场景提供一种可参考的工程方案。

\section{国内外研究现状}

近年来，基于大语言模型的自主智能体已成为人工智能领域的重要研究方向。Xi 等\upcite{xi2023agentsurvey} 系统梳理了 LLM Agent 的概念框架、能力构成与应用边界，Wang 等\upcite{wang2024agentsurvey} 则从智能体构建、任务执行、评估方法与应用场景等维度总结了大语言模型驱动自主智能体的发展脉络。围绕智能体的可视化编排与工程化实现，现有研究与产品实践主要集中在 Agent 编排平台与开发框架、Java 生态中的大模型应用支持、人工介入机制以及面向外部知识接入的检索增强生成等方向。

\subsection{AI Agent 编排平台与开发框架现状}

面向应用快速搭建的低代码平台发展迅速。Dify、Coze 等产品提供了可视化的 Agent 搭建界面；国内方面，百度、阿里、腾讯等厂商也推出了面向应用开发者的低代码构建能力。相关研究已经开始将 Dify 工作流能力用于测试项批量生成等场景\upcite{yin2025difytest}。这类平台的优势在于入门门槛低、节点配置直观、适合快速原型验证，但其运行时通常由平台自身托管，业务系统若要复用既有的数据模型、权限体系、审计规则和事务边界，往往仍需要额外的二次开发与适配。

在编程框架层面，Python 生态较为活跃。LangChain 是目前常用的 LLM 应用开发框架，提供丰富的模型接口、工具集成与记忆管理能力，但早期链式调用方式在表达复杂分支、循环和状态恢复时存在一定局限。为解决这一问题，LangGraph 引入图结构建模思想，将 Agent 的状态流转抽象为节点与边的组合，支持循环、分支、持久化执行和人工介入，成为当前智能体工作流编排的重要方案。微软研究院推出的 AutoGen\upcite{wu2023autogen} 则采用“对话即计算”范式，让多个 Agent 以对话方式完成任务分解与协作。在方法论层面，ReAct\upcite{yao2023react} 将“推理”与“行动”交织为统一决策范式，为工具调用型 Agent 提供了重要基础。上述框架为智能体编排提供了成熟参考，但对以 Java 和 Spring Boot 为主的企业后端系统而言，仍存在语言栈差异、运行环境隔离和业务系统集成等问题。

\subsection{Java 生态与人工审核现状}

在 Java 生态侧，Spring 体系在企业级应用开发中已经形成较成熟的工程实践，能够为大模型应用提供稳定的 Web 接口、依赖管理、事务处理和安全控制基础。但从系统编排角度看，现有 Java AI 应用框架更偏向 LLM 应用基础设施层，主要解决模型调用和外部工具接入问题，并不直接提供完整的可视化流程编排、图结构调度、执行状态持久化和人工审核恢复能力。Java 生态中与 LangGraph 类似的、面向 Agent 工作流的图导向编排实现相对较少，开发者若要在 Spring Boot 系统中实现节点调度、上下文传递、执行暂停和恢复，仍需要结合具体业务进行较多设计。

人工审核是企业业务流程中常见的控制机制，例如内容发布前审核、关键操作二次确认、自动生成结果复核和异常流程人工处理等。传统业务系统通常会将审批记录、操作人、审批意见和状态流转持久化，以便满足审计和追溯要求。相比之下，许多拖拉拽式 Agent 或工作流平台更重视自动化执行效率，缺少将“暂停—审核—修改—恢复”作为一等能力纳入运行时的设计。即使部分框架提供了人工介入入口，在与具体业务用户、权限控制、审核记录、并发审批和流程恢复语义结合时，也仍需要进一步工程化实现。

因此，在 Java 技术栈下设计一套既支持可视化编排，又能在执行过程中引入人工审核检查点的 Agent 编排系统，具有较强的实践意义。该类系统不应只关注模型调用结果，还需要管理工作流图结构、节点状态、执行上下文、审核队列和恢复策略，从而使智能体执行过程具备更好的可控性和可追溯性。

\subsection{知识检索增强与向量数据库现状}

在大模型应用中，检索增强生成（Retrieval-Augmented Generation, RAG）是缓解模型“幻觉”、引入外部知识的常见技术路径。Lewis 等\upcite{lewis2020rag} 最早提出了 RAG 范式，将参数化模型与外部检索器结合，使生成结果具备可追溯的知识依据。国内研究也对基于大型语言模型的检索增强生成技术进行了综述，梳理了 RAG 的基本流程、评估基准和未来挑战\upcite{liu2025ragsurvey,zhao2024ragoverview}。落到工程实现，向量数据库 Milvus\upcite{milvus2021} 提供了高维向量管理与近似最近邻检索能力，是支撑 RAG 应用的常用基础设施之一。对于需要接入业务文档或领域资料的 Agent 应用而言，在工作流执行过程中引入知识检索节点，有助于为 LLM 节点提供更具针对性的上下文信息。

综合以上分析，现有研究与产品已经在智能体框架、低代码编排和知识增强方面取得了较多进展，但在同时兼顾 Java 技术栈集成、可视化工作流编排、节点级人工审核、流程恢复和知识检索能力方面仍存在一定不足。因此，本文围绕上述问题设计并实现基于 Spring Boot 的 AI Agent 智能体编排系统，以验证相关能力组合在原型系统中的可行性。


\section{研究目标与内容}

本课题的研究目标是在 Spring Boot 技术体系下设计并实现一套 AI Agent 智能体编排系统，使系统能够支持智能体工作流的可视化配置、后端调度执行、人工审核干预、知识检索增强与基础安全认证。围绕上述目标，本文主要工作如下：

（1）调研国内外低代码 Agent 平台、Python 编排框架、Java 生态大模型应用和人工介入机制相关研究，分析智能体编排、工作流执行和人工审核机制结合方面的现状与不足，为系统的功能定位与技术选型提供依据。

（2）面向应用构建者和业务使用者等参与角色，分析 Agent 创建发布、工作流运行、人工审核、知识库管理和用户认证等典型业务流程，构建系统的功能性需求与非功能性需求模型。

（3）设计并实现基于 Spring Boot 的 AI Agent 智能体编排系统，覆盖 Agent 可视化编排、工作流执行与调度、人工审核检查点与流程恢复、知识文档存储与智能检索、用户认证与系统安全五项核心能力。

（4）围绕工作流运行时设计 DAG 调度、执行上下文传递、节点状态推进和 SSE 实时推送机制，并在关键节点引入人工检查点，使系统能够支持执行暂停、审批通过恢复、审批拒绝终止和审批记录追溯。

（5）面向核心功能与典型业务链路开展单元测试、功能测试、性能验证和部署验证，检查系统五项核心能力的功能闭环、关键链路可用性、基本性能表现和容器化部署可行性。

\section{论文组织结构}

本文围绕基于 Spring Boot 的 AI Agent 智能体编排系统展开，按照“背景分析—需求建模—概要设计—详细实现—测试部署—总结展望”的逻辑组织。

第 1 章为绪论，介绍课题背景与研究意义，梳理国内外研究现状，并明确本文的研究目标、主要工作和组织结构。

第 2 章为智能体编排系统需求分析，从用户角色、业务流程、功能需求和非功能需求等方面明确系统建设目标。

第 3 章为智能体编排系统概要设计，说明系统总体架构、功能模块划分、数据模型、核心状态流转和关键流程设计。

第 4 章为智能体编排系统详细设计与实现，重点阐述用户认证、Agent 管理、工作流调度、人工审核、知识库检索和前端界面等核心模块的实现过程。

第 5 章为智能体编排系统测试与部署，从单元测试、功能测试、性能表现和容器化部署等方面验证系统可用性与部署可行性。

最后为智能体编排系统总结与展望，总结全文工作，分析当前系统的不足，并对后续可扩展方向进行展望。
