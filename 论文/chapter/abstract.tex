\begin{abstractzh}

\noindent{\heiti\zihao{-4} 摘要：}大语言模型的发展推动人工智能应用从问答交互走向具备任务规划、工具调用和流程执行能力的 AI Agent。企业业务场景在降低应用构建门槛的同时，也要求执行过程可观察、关键结果可校验、操作记录可追溯。现有低代码编排平台和 Agent 开发框架能够支持流程搭建与自动执行，但在 Java 业务系统中的可视化编排、图结构调度和人工干预实践仍相对有限，许多工作流启动后连续执行至结束，缺少在关键节点暂停、审核、修改中间结果并恢复执行的机制。针对上述问题，本文设计并实现一套基于 Spring Boot 的 AI Agent 智能体编排系统。系统采用前后端分离与分层架构，前端基于 React、TypeScript 和 React Flow 构建可视化流程画布，后端按接口层、应用层、领域层和基础设施层组织核心逻辑；通过版本快照保存 Agent 配置，基于 DAG 调度推进节点状态，并利用 SSE 推送运行进度；同时在执行前后引入人工检查点，并通过审批记录持久化保证过程追溯。系统还集成知识文档存储与向量检索能力，并通过 Docker Compose 完成容器化部署验证。运行结果表明，系统实现了从 Agent 创建、流程发布、自动执行、人工审核到流程恢复的核心功能闭环，对 Java 生态下构建可控、可追踪的智能体执行平台具有一定工程实践价值。

\keywordszh{AI Agent；智能体编排；Spring Boot；工作流编排；人工审核检查点}

\end{abstractzh}

\begin{abstracten}

\noindent{\bfseries\zihao{-4} Abstract: }The development of large language models is driving artificial intelligence applications from question-answering interaction toward AI Agents capable of task planning, tool invocation and workflow execution. In enterprise scenarios, agent applications need not only low development barriers, but also observable execution, verifiable key results and traceable operations. Existing low-code orchestration platforms and agent development frameworks can support workflow construction and automated execution, but visual orchestration practices in Java-based business systems are still relatively limited. In addition, many workflows run continuously from start to finish once triggered, lacking mechanisms to pause at key nodes, conduct human review, modify intermediate results and resume execution. This makes it difficult to meet the requirements of high-risk business processes that combine automation with human control.

To address these issues, this thesis designs and implements an AI Agent orchestration system based on Spring Boot. The system targets application builders and business users, aiming to provide integrated capabilities for visual modeling, controllable execution, human review and execution tracking. It adopts a separated front-end/back-end architecture and a layered back-end design, organizing business logic into the interface, application, domain and infrastructure layers. The front end is built with React, TypeScript and React Flow to provide a visual workflow canvas. The core modules include agent management, workflow scheduling and execution, human review with workflow resumption, knowledge-base retrieval enhancement, and user authentication with security support.

In implementation, the system stores agent configurations through version snapshots, advances workflow node states through DAG scheduling, and pushes real-time execution progress to the front end through SSE. To address the lack of human control at key nodes, the workflow engine introduces two types of human checkpoints: before node execution and after node execution. When a workflow reaches a checkpoint, execution is paused and a pending review task is created. Reviewers can inspect the context, approve the workflow with modified intermediate results, or reject and terminate the workflow. A Redis-based queue, distributed locks, optimistic locking and persistent review records are used to ensure state consistency and process traceability under concurrent review scenarios. The system also supports knowledge enhancement and secure access through asynchronous document processing, Milvus vector retrieval and user authentication. The running results show that the system implements the core functional closed loop from agent creation, workflow publishing and automated execution to human review and workflow resumption, demonstrating practical engineering value for building controllable and traceable agent execution platforms in Java-based business systems.

\keywordsen{AI Agent; Agent Orchestration; Spring Boot; Workflow Orchestration; Human-in-the-loop}

\end{abstracten}
